# File Module

`file_module` — прикладной модуль `fastapi_template` для хранения файлов и их метаданных.

Модуль используется как единая точка для:

- загрузки файлов в S3-совместимое хранилище (MinIO);
- хранения метаданных файла в базе данных в модели `File`;
- скачивания файла через API с сохранением оригинального имени, включая кириллические;
- удаления файла из хранилища и записи из базы одной операцией.

Модуль опирается на инфраструктуру из `app.modules.system` и не дублирует общие CRUD- и DB-error-механизмы локально.

## Зависимость от `system`

`file_module` построен поверх `system` и повторно использует его общие примитивы:

- `Base` и `TimestampMixin` для ORM-модели;
- `CRUD` для операций с базой данных.

Предпочтительные импорты:

```python
from app.modules.system import Base, TimestampMixin, CRUD
```

## Основная модель

### `File`

Находится в [models/file.py](models/file.py).

`File` хранит метаданные о загруженном файле. Сам файл хранится в S3-бакете.

Ключевые поля:

- `id: int` — первичный ключ;
- `created_at: datetime` — дата и время создания записи;
- `updated_at: datetime` — дата и время последнего обновления;
- `link: str` — полный URL файла в бакете;
- `name: str` — оригинальное имя файла, сохранённое при загрузке (включая кириллические символы);
- `note: str | None` — произвольная заметка о файле.

Разделение ответственности:

- в бакете файл хранится под ключом `<uuid4_hex>.<extension>`, что исключает проблемы с кодировками в хранилище;
- оригинальное имя сохраняется в `name` и используется при скачивании через заголовок `Content-Disposition`.

## Pydantic-схемы

Схемы находятся в [schemas/file.py](schemas/file.py).

Основные DTO:

- `FileCreate` — внутренняя схема для `CRUD.create()`, принимает `link`, `name`, `note`;
- `FileRead` — схема ответа API, включает все поля модели.

`FileCreate` не используется напрямую клиентом. Хэндлер собирает её сам после загрузки файла в S3.

## S3-клиент

Находится в [services/s3_client.py](services/s3_client.py).

`S3Client` — асинхронный клиент для работы с S3-совместимым хранилищем на основе `aiobotocore`.

Методы:

- `create(file_obj, filename, content_type) → str` — стримит файл в бакет напрямую из `SpooledTemporaryFile`, без промежуточной копии в памяти; возвращает публичный URL;
- `read(link) → bytes` — скачивает файл по URL, возвращает байты;
- `delete(link) → None` — удаляет файл из бакета по URL.

Ключ в бакете генерируется как `uuid4().hex + расширение`, что обеспечивает уникальность и совместимость с любыми именами файлов.

Клиент берёт все параметры подключения из `app.core.config.settings`:

- `minio_endpoint`
- `minio_access_key`
- `minio_secret_key`
- `minio_bucket`
- `minio_secure`

В модуле доступен готовый синглтон:

```python
from app.modules.file_module.services import s3_client
```

## HTTP API

Router объявлен в [handlers.py](handlers.py) с префиксом `/files`.

### Endpoints

- `POST /api/files/` — загрузка файла;
- `GET /api/files/{id}` — скачивание файла;
- `DELETE /api/files/{id}` — удаление файла и записи.

### `POST /api/files/`

Принимает `multipart/form-data`:

- `file` — загружаемый файл (обязательно);
- `note` — заметка о файле (опционально).

Возвращает `FileRead` с `201 Created`.

Порядок действий:

1. нормализует имя: `file.filename or "unnamed"`;
2. передаёт `file.file` (`SpooledTemporaryFile`) напрямую в `s3_client.create()` — без `await file.read()`;
3. сохраняет метаданные в базе через `CRUD.create()`;
4. если сохранение в БД упало — удаляет только что загруженный файл из S3 и пробрасывает ошибку.

### `GET /api/files/{id}`

Возвращает файл как поток байт (`StreamingResponse`).

Порядок действий:

1. читает запись из БД через `CRUD.get()` — возвращает `404`, если не найдена;
2. скачивает байты из S3 через `s3_client.read()`;
3. определяет `Content-Type` по расширению имени файла через `mimetypes`;
4. формирует заголовок `Content-Disposition: attachment; filename*=UTF-8''<url-encoded-name>`, что позволяет браузерам и HTTP-клиентам корректно отобразить кириллическое имя.

### `DELETE /api/files/{id}`

Порядок действий:

1. читает запись из БД через `CRUD.get()` — возвращает `404`, если не найдена;
2. удаляет запись из БД (`session.delete` + `commit`) — транзакция, откат при сбое;
3. удаляет файл из S3 через `s3_client.delete()` — best-effort: сбой логируется, но не ломает ответ.

Возвращает `{"status": "ok"}`.

БД удаляется первой: если она упадёт, S3-файл остаётся цел. Если упадёт S3 — файл осиротеет в бакете, но битой ссылки в базе не будет.

## Правила изменений

Хорошие изменения в этом модуле:

- добавление полей метаданных, которые относятся к описанию файла;
- расширение `S3Client` новыми операциями с хранилищем;
- улучшение обработки ошибок при загрузке или удалении.

Нежелательные изменения:

- добавление бизнес-логики напрямую в `handlers.py`;
- дублирование CRUD-логики из `system` внутри модуля;
- хранение credentials напрямую в коде вместо `settings`.

## Практические заметки

- если меняется модель `File`, синхронно обновляйте `models`, `schemas` и создавайте новую миграцию;
- если меняется экспорт, обновляйте `file_module/__init__.py` и `app/modules/__init__.py`;
- если меняется поведение `S3Client`, обновляйте и `README.md`, и `AGENTS.md`;
- `Content-Type` при скачивании определяется по `name`, а не по `link`, поэтому важно сохранять имя файла с расширением.
